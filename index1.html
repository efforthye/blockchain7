style : 스타일은 속성이고, 그 태그의 크기나 외곽선, 위치, 배경 등을 설정한다. style 속성을 사용하는 것은 기초적인 방법이었다. 예전에 웹 페이지는 html파일로만 주고 받았다. 점점 발전하다보니 html파일이 너무 커져 보기 어려워서 나온 것이 css 파일이다. CSS : Cascading Style Sheets로 HTML파일을 외부로 빼서 불러올 수 있게 하고 보기도 편해지게 했다.

style에서 너비나 높이나 숫자를 이용한 수치들에 대한 단위 : px, %, em, rem, vw, vh까지 있다. 이놈들을 설명하기 위해서는 부모라는 개념이 꼭 필요하다고 한다.

px : 그냥 픽셀, 그림판의 작은 한 칸, 화소

% : 부모를 기준으로 한 퍼센트, 100%는 부모의 너비와 같다는 뜻이다.

조금 특이한 게 em과 rem인데

em : 글씨 크기를 기준으로 한다. 부모의 글씨 크기를 따름

rem : 글씨 크기를 기준으로 한다. 부모가 아닌 HTML의 글씨크기를 따르며 rem은 반응형 웹에서 생각보다 많이 사용한다.(중요) rem은 모바일이나 pc에 한번에 만들 때

ex) 만약 글씨 크기가 16px이면 1em, 1rem은 16px이다.
글씨 크기가 32px, 1rem = 1em = 32px, 0.5rem = 16px이다

vw, vh : viewport, 브라우저의 크기나 스마트폰의 크기를 기준으로 %를 따진다. 100vw : 화면의 너비, 100vh: 화면의 높이이다.

width : 그냥 영역의 너비이다. max-width : 영역의 최대 너비이다. min-width : 영역의 최소 너비이다. %, vw, vh은 기준이 있고, 그 기준의 비율로 너비를 설정한다. 그런데 어떠한 내용을 넣었을 때 최소 100px은 있어야 제대로 뜨거나 할때 min-width를 쓴다. 어떤 내용을 넣었는데 너무 크면 안돼, 1000px이상 넘어가면 안된다 할 때 max-width를 쓴다. 반응형 웹을 할 때 min-width, max-width를 많이 사용할 것이다.(중요) 네이버에서 줄였을 때 너비가 딱 맞는 이유는 max-width 때문이다. 브라우저가 줄어들지 않는 이유는 브라우저 자체에 최소크기가 있기 때문이다.
height : 높이 max-height : 영역의 최대 높이 min-height : 영역의 최소 높이 이 둘은 거의 사용할 일 없지만 이미지를 조절할 때 사용한다. 왜냐면 세로 스크롤은 많은데 가로 스크롤은 별로 사용할 일이 없기 때문. (이해하기) 이미지에 대한 style을 할 때 다시 설명해주신다고 한다.
